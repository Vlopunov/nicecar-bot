# NICECAR CENTER — Telegram Bot + Mini App + Admin Panel

## ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА

Разработай полнофункциональную Telegram-экосистему для детейлинг центра **NICECAR Center** (официальный центр бренда **Krytex** в Минске). Система состоит из трёх связанных компонентов:

1. **Telegram Bot** — клиентский бот для записи, каталога услуг и FAQ
2. **Telegram Mini App (WebApp)** — полноценное клиентское приложение с каталогом, записью, личным кабинетом и программой лояльности
3. **Telegram Mini App (Admin)** — админ-панель для управления бизнесом (расписание, CRM, рассылки)

### Backend
- **Python 3.11+**
- **aiogram 3.x** — Telegram Bot Framework
- **FastAPI** — REST API для Mini App и админки

- **PostgreSQL** — основная БД

## ДЕТАЛЬНЫЕ ТРЕБОВАНИЯ

### 1. TELEGRAM BOT (aiogram 3.x)

#### 1.1 Команда /start
- Приветствие с логотипом NiceCar
- Автоматическая регистрация пользователя в БД (telegram_id, username, first_name, last_name, phone если доступен)
- Если пришёл по реферальной ссылке `/start ref_{user_id}` — связать реферала
- Главное меню (ReplyKeyboardMarkup):
  ```
  📋 Услуги и цены    |  📅 Записаться
  📸 Наши работы       |  ❓ FAQ
  👤 Личный кабинет    |  📞 Связаться
  ```

#### 1.2 Каталог услуг (📋 Услуги и цены)
- InlineKeyboard с категориями услуг
- При выборе категории — список услуг с ценами
- Для услуг с ценами по классам авто — показать таблицу (message с форматированием)
- Для пакетных услуг (антигравийка) — описание пакетов MINI/STANDART/PREMIUM/INDIVIDUAL
- Кнопка "Записаться на эту услугу" → открывает Mini App на странице записи с предвыбранной услугой

#### 1.3 Запись на услугу (📅 Записаться)
- Кнопка WebApp → открывает Mini App на странице записи
- Альтернативный flow через бота (для тех кто не хочет Mini App):
  1. Выбор категории → услуги → конкретная услуга (InlineKeyboard)
  2. Выбор класса авто (если применимо)
  3. Ввод марки/модели авто (текстом)
  4. Выбор даты (InlineCalendar, показывать только рабочие дни, не раньше завтра)
  5. Выбор времени из свободных слотов
  6. Подтверждение записи (summary всех данных)
  7. Отправка уведомления менеджеру

#### 1.4 Отправка фото для расчёта (📸)
- Пользователь отправляет фото авто
- Бот сохраняет фото и пересылает в админ-чат с данными клиента
- Ответ: "Спасибо! Менеджер рассчитает стоимость и свяжется с вами в течение 30 минут (в рабочее время)"
- Менеджер может ответить клиенту прямо через бота (reply в админ-чате → бот пересылает клиенту)

#### 1.5 FAQ (❓)
- Inline-кнопки по темам (загружаются из БД, управляются через админку)
- Предзаполненные FAQ из сайта:
  - Антигравийная плёнка (10+ вопросов с сайта)
  - Химчистка
  - Мойка
  - Общие вопросы (режим работы, адрес, парковка, рассрочка)
- Кнопка "Задать свой вопрос" → пересылка в админ-чат

#### 1.6 Личный кабинет (👤)
- Кнопка WebApp → открывает Mini App на странице профиля
- Краткая инфо в боте: имя, кол-во визитов, бонусный баланс, реферальная ссылка

#### 1.7 Связаться (📞)
- Телефон: +375 (29) 664-94-87
- Адрес с ссылкой на карту
- Кнопка "Написать менеджеру" (пересылка в админ-чат)
- Viber / Telegram контакт

#### 1.8 Уведомления клиентам (от бота)
- Напоминание за 24 часа до записи
- Напоминание за 2 часа до записи
- Статус записи (подтверждена / в работе / завершена)

**Логика слотов (SlotService):**
```python
# Генерация слотов на день:
# 1. Получить все посты (WorkPost), активные
# 2. Получить все записи на этот день
# 3. Для каждого временного слота (с шагом 30 мин):
#    - Посчитать сколько постов заняты в этот момент
#    - Если есть хотя бы 1 свободный пост — слот доступен
# 4. Учесть длительность услуги: слот доступен только если пост свободен на всю длительность
```

    price_final: Decimal (nullable)     # итоговая цена

    created_at: datetime
    updated_at: datetime
    reminder_24h_sent: bool (default False)
    reminder_2h_sent: bool (default False)

# PortfolioItem
class PortfolioItem(Base):

    category_id: int (FK → ServiceCategory.id, nullable)
    service_id: int (FK → Service.id, nullable)
    image_url: str
    image_before_url: str (nullable)   # фото "до"
    car_brand: str (nullable)
    car_model: str (nullable)
    description: text (nullable)

### 8. ВАЖНЫЕ ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

1. **Telegram initData валидация** — ОБЯЗАТЕЛЬНО проверять HMAC-подпись для всех API-запросов из Mini App. Использовать стандартный алгоритм Telegram (HMAC-SHA256)
2. **Async everywhere** — вся работа с БД, API, ботом через async/await
3. **Timezone** — все даты/времена в Europe/Minsk (UTC+3)
4. **Миграции** — Alembic, initial migration со всеми таблицами
5. **Error handling** — глобальный обработчик ошибок, логирование
6. **Rate limiting** — для API и бота
7. **CORS** — настроить для Mini App домена
8. **Graceful shutdown** — корректное завершение бота и воркеров
9. **Docker** — docker-compose для локальной разработки (postgres + redis + app)
10. **Health check** — эндпоинт /health для Railway
11. **Загрузка изображений** — сохранять в /uploads или использовать object storage, отдавать через API
12. **Instagram парсинг** — через instaloader или подобную библиотеку, с обработкой rate limits
